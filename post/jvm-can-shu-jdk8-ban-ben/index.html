Option and Default Value Description

-agentpath 加载由绝对路径名指定的本地代理程序库
-client 选择Java HotSpot客户端VM
-Dproperty 设置系统属性值
-d32 在32位环境中运行应用程序
-d64 在64位环境中运行应用程序
-disableassertions/-da 禁用断言
-disablesystemassertions/-dsa 禁用所有系统类中的断言
-enableassertions/-ea 启用断言
-enablesystemassertions/-esa 在所有系统类中启用断言
-help 显示该java命令的使用情况信息
-jar 执行封装在JAR文件中的程序
-javaagent 加载指定的Java编程语言代理
-server 选择Java HotSpot Server VM
-showversion 显示版本信息并继续执行应用程序 HotSpot Server VM
-splash 用imgname指定的图像显示启动画面
-verbose 显示有关每个加载的类的信息
-verbose:class 显示有关每个加载的类的信息
-verbose:GC 显示有关每个垃圾回收（GC）事件的信息。
-verbose:JNI 显示有关使用本地方法和其他Java Native Interface（JNI）活动的信息。
-version 显示版本信息，然后退出
-version 显示版本信息，然后退出

非标准选项

Option and Default Value Description
-X 显示所有可用-X选项的帮助
-Xbatch 禁用后台编译
-Xbootclasspath:path 指定由冒号（:)分隔的目录，JAR文件和ZIP存档的列表，以搜索引导类文件
-Xbootclasspath /a:path 指定由冒号（:)分隔的目录，JAR文件和ZIP存档的列表，以附加到默认引导程序类路径的末尾
-Xbootclasspath /p:path 指定由冒号（:)分隔的目录，JAR文件和ZIP存档的列表，以便将其前置到默认引导程序类路径的前面。
-Xcheck：JNI 对Java本地接口（JNI）功能执行额外的检查
-Xcomp 首次调用时强制编译方法
-Xdebug 什么也没做。提供向后兼容性。
-Xdiag 显示附加的诊断信息。
-Xfuture 启用严格的类文件格式检查，以严格遵守类文件格式规范。鼓励开发人员在开发新代码时使用此标志，因为更严格的检查将成为未来版本的默认检查
-Xint 以仅解释模式运行应用程序
-Xinternalversion 显示比-version选项更详细的JVM版本信息，然后退出。
-Xloggc:filename 设置详细GC事件信息应该重定向到日志的文件
-Xmaxjitcodesize=size 定JIT编译代码的最大代码高速缓存大小（以字节为单位）
-Xmixed 除了编译为本地代码的热门方法之外，由解释器执行所有的字节码
-Xmn=size 设置年轻一代的堆的初始和最大大小（以字节为单位）
-Xms=size 设置堆的初始大小（以字节为单位）。该值必须是1024的倍数，大于1 MB。
-Xmx=size 指定内存分配池的最大大小（以字节为单位）。该值必须是1024的倍数并且大于2 MB。
-Xnoclassgc 禁用类的垃圾回收（GC）
-Xprof 分析正在运行的程序并将分析数据发送到标准输出
-Xrs 减少JVM对操作系统信号的使用
-Xshare 设置类数据共享（CDS）模式
-XshowSettings:category 显示设置并继续。
-Xss 设置线程堆栈大小（以字节为单位。
-Xusealtsigs 使用替代信号，而不是SIGUSR1和SIGUSR2对JVM内部信号
-Xverify 设置字节码验证器的模式
-Xverify 设置字节码验证器的模式

高级运行时选项

Option and Default Value Description
-XX：+ CheckEndorsedAndExtDirs 启用该选项以防止java命令运行Java应用程序
-XX：+ DisableAttachMechanism 启用禁用允许工具附加到JVM的机制的选项
-XX：ErrorFile=filename 指定发生不可恢复错误时写入错误数据的路径和文件名
-XX：+ FailOverToOldVerifie 当新类型检查器失败时，启用自动故障转移到旧的检验器
-XX：+ FlightRecorder 在应用程序的运行时期间启用Java Flight Recorder（JFR）
-XX：-FlightRecorder 在应用程序运行期间禁用Java Flight Recorder（JFR）
-XX：FlightRecorderOptions = parameter = value 设置控制JFR行为的参数
-XX：LargePageSizeInBytes =size 设置用于Java堆的大页面的最大大小
-XX：MaxDirectMemorySize = size 设置新I / O（java.nio包）直接缓冲区分配的最大总大小
-XX：NativeMemoryTracking = model 指定跟踪JVM本机内存使用情况的模式
-XX：ObjectAlignmentInBytes = 对齐 设置Java对象的内存对齐方式（以字节为单位）
-XX：OnError = 字符串 设置自定义命令或一系列分号分隔的命令，以便在发生不可恢复的错误时运行
-XX：OnOutOfMemoryError = 字符串 设置一个自定义命令或一系列分号分隔的命令，以便在OutOfMemoryError第一次抛出异常时运行
-XX：+ PerfDataSaveToFile 如果启用，则在Java应用程序退出时保存jstat（1）二进制数据
-XX：+ PrintCommandLineFlags 启用打印出现在命令行上的人体工程学选择的JVM标志
-XX：+ PrintNMTStatistics 启用本地内存跟踪后，在JVM退出时启用收集的本地内存跟踪数据打印
-XX：+ RelaxAccessControlCheck 减少验证器中的访问控制检查的数量
-XX：+ ResourceManagement 在应用程序的运行时期间启用资源管理。
-XX：ResourceManagementSampleInterval = value 设置控制资源管理测量的采样间隔的参数。
-XX：SharedArchiveFile = path 指定类数据共享（CDS）存档文件的路径和名称。
-XX：SharedClassListFile = file_name 指定包含要存储在类数据共享（CDS）存档中的类文件名称的文本文件
-XX：+ ShowMessageBoxOnError 当JVM遇到不可恢复的错误时，启用显示对话框。
-XX：StartFlightRecording = parameter = value 为Java应用程序启动JFR记录。
-XX：ThreadStackSize = size 设置线程堆栈大小（以字节为单位）。
-XX：+ TraceClassLoading 启用类加载跟踪。
-XX：+ TraceClassLoadingPreorder 以所引用的顺序启用所有加载的类的跟踪。
-XX：+ TraceClassResolution 启用对常量池分辨率的跟踪。
-XX：+ TraceClassUnloading 在卸载类时启用跟踪。
-XX：+ UnlockCommercialFeatures 启用商业功能。
-XX：+ UseAltSigs 使得能够使用替代信号，而不是SIGUSR1和SIGUSR2用于JVM内部信号。
-XX：+ UseAppCDS 启用应用程序类数据共享（AppCDS）。
-XX：-UseBiasedLocking 禁用使用偏置锁定。
-XX：-UseCompressedOops 禁用压缩指针的使用。
-XX：+UseHugeTLBFS 这个选项对于Linux来说是相当于指定的-XX:+UseLargePages。
-XX：+UseLargePages 启用大页面内存的使用。
-XX：+UseMembar 在线程状态转换中启用membars。
-XX：+UsePerfData 启用该perfdata功能。
-XX：+UseTransparentHugePages 在Linux上，可以使用可动态增长或缩小的大页面。
-XX：+AllowUserSignalHandlers 允许应用程序安装信号处理程序。

高级JIT编译器选项

Option and Default Value Description
-XX：+ AggressiveOpts 启用积极的性能优化功能，预计这些功能将在即将发布的版本中成为默认功能
-XX：AllocateInstancePrefetchLines = line 设置要在实例分配指针之前预取的行数
-XX：AllocatePrefetchDistance = size 设置对象分配的预取距离的大小（以字节为单位）。
-XX：AllocatePrefetchInstr = 指令 将预取指令设置为在分配指针之前预取。
-XX：AllocatePrefetchLines = line 通过使用编译代码中生成的预取指令，设置最后一次分配对象后要加载的高速缓存行数。
-XX：AllocatePrefetchStepSize = size 设置顺序预取指令的步长（以字节为单位）。
-XX：AllocatePrefetchStyle = style 为预取指令设置生成的代码样式。
-XX：+ BackgroundCompilation 启用后台编译。
-XX：CICompilerCount = thread 设置要用于编译的编译器线程的数量。
-XX：CodeCacheMinimumFreeSpace = size 设置编译所需的最小可用空间（以字节为单位）。
-XX：CompileCommand = command，method [，option ] 指定要在方法上执行的命令。
-XX：CompileCommandFile = 文件名 设置从中读取JIT编译器命令的文件。
-XX：CompileOnly = 方法 设置编译应该限制的方法列表（用逗号分隔）。
-XX：CompileThreshold = invocations 在编译之前设置解释的方法调用的数量。
-XX：+ DoEscapeAnalysis 允许使用转义分析。
-XX：InitialCodeCacheSize = size 设置初始代码缓存大小（以字节为单位）。
-XX：+内嵌 启用方法内联。
-XX：InlineSmallCode = size 设置应该内联的已编译方法的最大代码大小（以字节为单位）。
-XX：+ LogCompilation 使编译活动能够记录到hotspot.log当前工作目录中指定的文件。
-XX：MaxInlineSize = 大小 设置内联方法的最大字节码大小（以字节为单位）。
-XX：MaxNodeLimit = 节点 设置单个方法编译期间使用的最大节点数。
-XX：MaxTrivialSize = 大小 设置内联方法的最大字节码大小（以字节为单位）。
-XX：+ OptimizeStringConcat 启用String串联操作的优化。
-XX：+ PrintAssembly 通过使用外部disassembler.so库启用打印字节码和本地方法的汇编代码。
-XX：+ PrintCompilation 每次编译方法时，都会通过向控制台输出消息来启用JVM的详细诊断输出
-XX：+ PrintInlining 启用内联决策的打印
-XX：ReservedCodeCacheSize = size 设置JIT编译代码的最大代码缓存大小（以字节为单位）
-XX：RTMAbortRatio = abort_ratio RTM异常中止率是指所有执行的RTM事务的百分比
-XX：RTMRetryCount = number_of_retries RTM锁定代码将被重试，当它被中止或繁忙时，在返回到正常的锁定机制之前由该选项指定的次数
-XX：-TieredCompilation 禁用分层编译的使用
-XX：+ UseAES 禁用分层编译的使用
-XX：+ UseAESIntrinsics UseAES和UseAESIntrinsics标志默认启用，仅支持Java HotSpot Server VM 32位和64位
-XX：+ UseCodeCacheFlushing 在关闭编译器之前启用代码缓存刷新
-XX：+ UseCondCardMark 在更新卡表之前，检查卡是否已被标记。
-XX：+ UseRTMDeopt 根据中止比率自动调整RTM锁定
-XX：+ UseRTMLocking 为所有虚拟锁生成受限制的事务内存（RTM）锁定代码，使用正常的锁定机制作为后备处理程序
-XX：+ UseSHA 为SPARC硬件的SHA加密哈希函数启用基于硬件的内在函数
-XX：+ UseSHA1Intrinsics 为SHA-1密码散列函数启用内部函数
-XX：+ UseSHA256Intrinsics 为SHA-224和SHA-256密码哈希函数启用内部函数。
-XX：+ UseSHA512Intrinsics 为SHA-384和SHA-512加密哈希函数启用内部函数。
-XX：+ UseSuperWord 启用标量操作转换为超字节操作。

高级可维护性选项

Option and Default Value Description
-XX：+ ExtendedDTraceProbes 启用dtrace影响性能的其他工具探针
-XX：+ HeapDumpOnOutOfMemory 在java.lang.OutOfMemoryError引发异常时，使用堆分析程序（HPROF）启用将Java堆转储到当前目录中的文件
-XX：HeapDumpPath = path 设置用于在设置-XX:+HeapDumpOnOutOfMemoryError选项时写入由堆分析器（HPROF）提供的堆转储的路径和文件名
-XX：LogFile = path 设置写入日志数据的路径和文件名
-XX：+ PrintClassHistogram 在Control+C事件（SIGTERM）之后启用类实例直方图的打印
-XX：+ PrintConcurrentLocks java.util.concurrent在Control+C事件（SIGTERM）后启用锁打印
-XX：+ UnlockDiagnosticVMOptions java.util.concurrent在Control+C事件（SIGTERM）后启用锁打印

高级垃圾收集选项

Option and Default Value Description
-XX：+ AggressiveHeap 启用Java堆优化
-XX：+ AlwaysPreTouch 在JVM初始化期间，可以触摸Java堆上的每个页面
-XX：+ CMSClassUnloadingEnabled 使用并发标记清除（CMS）垃圾回收器时启用类卸载
-XX：CMSExpAvgFactor = 百分比 设置计算并发收集统计数据的指数平均值时用于对当前样本进行加权的时间百分比（0到100）
-XX：CMSInitiatingOccupancyFraction = 百分比 设置启动CMS收集周期的旧代占有率（0到100）的百分比
-XX：+ CMSScavengeBeforeRemark 在CMS注释步骤之前启用扫描尝试
-XX：CMSTriggerRatio = 百分比 设置-XX:MinHeapFreeRatioCMS收集周期开始之前分配的百分比（0到100）。默认值设置为80％。
-XX：ConcGCThreads = 线程 设置用于并发GC的线程数。缺省值取决于JVM可用的CPU数量。
-XX：+ DisableExplicitGC 启用禁用呼叫处理的选项System.gc()。
-XX：+ ExplicitGCInvokesConcurrent 启用使用System.gc()请求调用并发GC 。
-XX：+ ExplicitGCInvokesConcurrentAndUnloadsClasses 通过System.gc()在并发GC循环期间使用请求和卸载类来启用并发GC的调用 。
-XX：G1HeapRegionSize = size 设置在使用垃圾优先（G1）收集器时Java堆被细分的区域的大小。
-XX：+ G1PrintHeapRegions 启用打印有关分配哪些区域以及哪些区域由G1收集器回收的信息。
-XX：G1ReservePercent = 百分比 设置保留为假天花板的堆的百分比（0到50），以减少G1收集器升级失败的可能性。
-XX：InitialHeapSize = size 设置内存分配池的初始大小（以字节为单位）。
-XX：InitialSurvivorRatio = 比率 设置吞吐量垃圾回收器使用的初始存活空间比率。
-XX：InitiatingHeapOccupancyPercent = 百分比 设置启动并发GC循环的堆占用百分比（0到100）。
-XX：MaxGCPauseMillis = 时间 设置最大GC暂停时间（以毫秒为单位）的目标。
-XX：MaxHeapSize = 大小 设置内存分配池的最大大小（以字节为单位）
-XX：MaxHeapFreeRatio = 百分比 设置GC事件后允许的最大空闲堆空间百分比（0到100）
-XX：MaxMetaspaceSize = size 设置可以为类元数据分配的最大本机内存量
-XX：MaxNewSize = 大小 设置年轻一代堆的最大大小（以字节为单位）
-XX：MaxTenuringThreshold = 阈值 设置自适应GC大小的最大使用期限
-XX：MetaspaceSize = 大小 设置首次超出时将触发垃圾回收的已分配类元数据空间的大小。
-XX：MinHeapFreeRatio = 百分比 GC事件后设置空闲堆空间的最小允许百分比（0到100）。
-XX：NewRatio = 比率 设置年轻代和老年代之间的比例。
-XX：NewSize = 大小 设置年轻一代堆的初始大小。
-XX：ParallelGCThreads = 线程 设置在年轻代和老年代中用于并行垃圾收集的线程数。
-XX：+ ParallelRefProcEnabled 启用并行参考处理。
-XX：+ PrintAdaptiveSizePolicy 启用打印有关自适应生成大小的信息。
-XX：+ PrintGC 允许在每个GC上打印消息。
-XX：+ PrintGCApplicationConcurrentTime 允许打印自上次暂停以来经过的时间。
-XX：+ PrintGCApplicationStoppedTime 可以打印暂停（例如GC暂停）持续的时间。
-XX：+ PrintGCDateStamps 在每个GC上打印日期戳记。
-XX：+ PrintGCDetails 可在每个GC上打印详细的消息。
-XX：+ PrintGCTaskTimeStamps 为每个GC工作线程任务启用打印时间戳。
-XX：+ PrintGCTimeStamps 在每个GC上打印时间戳。
-XX：+ PrintStringDeduplicationStatistics 打印详细的重复数据删除统计信息。
-XX：+ PrintTenuringDistribution 启用时代年龄信息的打印。
-XX：+ ScavengeBeforeFullGC 在每个完整的GC之前启用年轻一代的GC。
-XX：SoftRefLRUPolicyMSPerMB = 时间 设置在最后一次引用之后，软堆中可轻松访问的对象在堆上保持活动的时间。
-XX：StringDeduplicationAgeThreshold = 阈值 String达到指定年龄的对象被认为是重复数据删除的候选对象。
-XX：SurvivorRatio = 比率 设置伊甸园空间大小和幸存者空间大小之间的比率。
-XX：TargetSurvivorRatio = 百分比 设置年轻垃圾回收后使用的生存空间（0到100）的所需百分比。
-XX：TLABSize = size 设置线程本地分配缓冲区（TLAB）的初始大小（以字节为单位）
-XX：+ UseAdaptiveSizePolic 允许使用自适应生成大小
-XX：+ UseCMSInitiatingOccupancyOnly 启用占用值作为启动CMS收集器的唯一标准
-XX：+ UseConcMarkSweepGC 为旧一代启用CMS垃圾收集器
-XX：+ UseG1GC 启用垃圾优先（G1）垃圾回收器
-XX：+ UseGCOverheadLimit 在引发OutOfMemoryError异常之前，允许使用限制JVM花费在GC上的时间比例的策略
-XX：+ UseNUMA 通过增加应用程序对低延迟内存的使用，在非统一内存架构（NUMA）的计算机上启用应用程序的性能优化
-XX：+ UseParallelGC 启用并行扫描垃圾回收器（也称为吞吐量收集器），通过利用多个处理器来提高应用程序的性能。
-XX：+ UseParallelOldGC 为完整的GC启用并行垃圾回收器
-XX：+ UseParNewGC 在年轻一代中允许使用并行线程进行收集
-XX：+ UseSerialGC 启用串行垃圾收集器
-XX：+ UseSHM 在Linux上，使JVM能够使用共享内存来设置大页面。
-XX：+ UseStringDeduplication 启用字符串重复数据删除。
-XX：+ UseTLAB 在新一代空间中启用线程局部分配块（TLAB）。
-XX：+ UseTLAB 在新一代空间中启用线程局部分配块（TLAB）。
